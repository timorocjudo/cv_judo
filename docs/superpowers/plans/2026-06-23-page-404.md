# Page 404 Personnalisée IpponId — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Créer `app/not-found.tsx` avec wordmark, "404" stylisé, wording judo, deux CTAs, fade-in Framer Motion, et auto-focus du champ de recherche via `?focus=search`.

**Architecture:** `app/not-found.tsx` est un Server Component minimal (metadata + import). Tout le contenu visuel est dans `components/NotFoundContent.tsx` (client component, Framer Motion). `FooterLegal` est déjà rendu par `FooterLegalConditional` dans `app/layout.tsx` sur toutes les routes non-`/` — il apparaîtra automatiquement sur la 404 sans ajout de code. `SearchAutocomplete` reçoit `useSearchParams()` + `inputRef` pour l'auto-focus.

**Tech Stack:** Next.js 14 App Router, Framer Motion 12, Tailwind CSS 3 (arbitrary values), TypeScript

## Global Constraints

- `not-found.tsx` reste Server Component — pas de `'use client'`, pas de `usePathname`, pas de data fetching
- `FooterLegal` n'est PAS à importer dans `not-found.tsx` — déjà géré par `FooterLegalConditional` dans le layout global
- `useReducedMotion()` de Framer Motion doit être respecté : si `true`, pas d'animation
- Tokens Tailwind exclusivement sauf pour le `fontSize` du "404" (hors palette, `style` inline avec `clamp()` acceptable)
- `useSearchParams()` requiert `<Suspense>` wrapper dans les Server Components parents — `HeroSection` doit être mis à jour
- Mobile-first : boutons empilés sur mobile (`flex-col`), côte à côte sur `sm:` (`sm:flex-row`)

### Task 3: Auto-focus `?focus=search` dans `SearchAutocomplete`

**Files:**
- Modify: `components/SearchAutocomplete.tsx`
- Modify: `components/landing/HeroSection.tsx`

**Interfaces:**
- Consumes: `useSearchParams` de `next/navigation` (nouvel import), `useRef` déjà importé de `react`
- Produces: `SearchAutocomplete` — même API externe, + auto-focus si `searchParams.get('focus') === 'search'`

- [ ] **Step 1: Modifier `components/SearchAutocomplete.tsx`**

Ajouter `useSearchParams` à l'import next/navigation (nouvelle ligne 5) :
```tsx
import { useSearchParams } from 'next/navigation'
```

Ajouter après la ligne `const debounceRef = useRef<...>(null)` (ligne 30) :
```tsx
const inputRef = useRef<HTMLInputElement>(null)
const searchParams = useSearchParams()

useEffect(() => {
  if (searchParams.get('focus') === 'search') {
    inputRef.current?.focus()
  }
}, [searchParams])
```

Sur le `<input>` (ligne ~113), ajouter `ref={inputRef}` :
```tsx
<input
  ref={inputRef}
  type="text"
  value={query}
  onChange={handleChange}
  onKeyDown={handleKeyDown}
  placeholder={placeholder}
  autoComplete="off"
  role="combobox"
  aria-autocomplete="list"
  aria-expanded={isOpen}
  aria-haspopup="listbox"
  className="flex-1 border-none outline-none focus:ring-0 px-4 py-3 text-on-surface placeholder:text-outline bg-transparent"
/>
```

```tsx
      </div>

      <div className="max-w-4xl mx-auto relative z-10">
        <h1 className="font-montserrat text-headline-lg-mobile md:text-headline-lg font-black text-primary mb-6 leading-tight">
          Ton palmarès judo mérite<br className="hidden md:block" /> sa propre page
        </h1>
        <p className="text-body-lg text-on-surface-variant mb-10 max-w-2xl mx-auto">
          Crée ta page CV de judoka gratuitement. Partage ton parcours, tes grades et tes victoires avec ton URL personnalisée.
        </p>

        <Suspense>
          <SearchAutocomplete className="max-w-2xl mx-auto" />
        </Suspense>
      </div>
    </section>
  )
}
```

**Pourquoi `<Suspense>` sans fallback :** `SearchAutocomplete` s'affiche instantanément côté client (c'est un composant client dans un Server Component). Le `<Suspense>` sans fallback affiche `null` le temps que le client hydrate, puis le composant apparaît — comportement identique à l'actuel puisque le composant est déjà `'use client'`. Ajouter `fallback={<div className="h-[58px]" />}` évite un layout shift si souhaité (optionnel).
