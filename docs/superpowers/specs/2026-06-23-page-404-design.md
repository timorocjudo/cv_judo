# Spec — Page 404 personnalisée IpponId

_Date : 2026-06-23_

## Objectif

Créer une page 404 cohérente avec la charte IpponId qui transforme une route inconnue en opportunité de garder l'utilisateur sur le site (recherche d'un judoka ou retour accueil).

## Fichier cible

`app/not-found.tsx` — détecté automatiquement par Next.js App Router pour toutes les routes non trouvées.

## Architecture

### Composants réutilisés

| Composant | Usage |
|---|---|
| `components/layout/LogoLink.tsx` | Wordmark "IpponId" en haut de la page |
| `components/FooterLegal.tsx` | Footer légal en bas |
| `components/SearchAutocomplete.tsx` | Modifié pour gérer `?focus=search` |

### Nouveau fichier

- `app/not-found.tsx` — Server Component (pas d'async, pas d'auth)

### Modification

- `components/SearchAutocomplete.tsx` — ajout de `useSearchParams()` + `useRef` pour auto-focus si `?focus=search` est présent dans l'URL

- Fond blanc (`bg-white`)

- Sur mobile : boutons empilés verticalement (`flex-col`), full-width (`w-full`)
- Sur desktop : boutons côte à côte (`sm:flex-row`), largeur auto

## Animation

Composant client `NotFoundContent` (wrappé dans `not-found.tsx`) :

```tsx
const shouldReduceMotion = useReducedMotion()

<motion.div
  initial={shouldReduceMotion ? false : { opacity: 0, y: 16 }}
  animate={{ opacity: 1, y: 0 }}
  transition={{ duration: 0.3, ease: 'easeOut' }}
>
  {/* contenu */}
</motion.div>
```

Pattern identique à `FadeInOnScroll` mais avec `animate` (immédiat) plutôt que `whileInView` (scroll-triggered).

## Comportement `?focus=search`

Dans `SearchAutocomplete` :

```tsx
const searchParams = useSearchParams()
const inputRef = useRef<HTMLInputElement>(null)

useEffect(() => {
  if (searchParams.get('focus') === 'search') {
    inputRef.current?.focus()
  }
}, [searchParams])
```

L'`<input>` existant reçoit `ref={inputRef}`. Pas d'autre changement de comportement.

**Note :** `useSearchParams()` nécessite que `SearchAutocomplete` soit wrappé dans un `<Suspense>` dans ses composants parents. À vérifier — si `HeroSection` est déjà dans un contexte correct (il l'est, c'est un Server Component parent), un `<Suspense>` minimal suffira.

## Flux utilisateur

1. L'utilisateur tape `/slug-inexistant` → Next.js sert `not-found.tsx` avec status 404
2. La page s'affiche avec fade-in (300ms)
3. CTA "Rechercher un judoka" → navigue vers `/?focus=search` → `SearchAutocomplete` reçoit le focus automatiquement
4. CTA "Retour à l'accueil" → navigue vers `/`

## Ce qui est hors scope

- Internationalisation (site 100% FR)
- Tracking analytics de la 404 (pas de GA/Plausible actuellement)
- Animation du "404" (chiffre qui bounce, etc.) — fade-in simple suffit
